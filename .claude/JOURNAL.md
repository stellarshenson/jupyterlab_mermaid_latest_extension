4. **Task - Fix broken-image regression** (v1.0.5): Reworked the monkey-patch so every diagram renders again instead of showing broken images<br>
   **Result**: Published 1.0.4 rendered all diagrams as broken images. Root cause: `patchMermaidManager` reimplemented `renderFigure`/`renderSvg` and built its own `image/svg+xml` data URI, but skipped JupyterLab's void-element fixup - mermaid's htmlLabels emit unclosed `<br>` inside `foreignObject`, which is invalid XML in an `<img>` data URI, so the browser drew the broken-image icon. Earlier Playwright "verification" only checked `img.src` contained the SVG text, not that it parsed. Fix: override ONLY `getMermaid()` and `getMermaidVersion()` on the live manager, letting JupyterLab's proven `renderSvg`/`cleanMermaidSvg`/`makeMermaidFigure` run our bundled mermaid 11.15.0; matched the built-in init config (`securityLevel`, `maxEdges`, `--jp-ui-font-family`). Also fixed version detection by reading `mermaid/package.json` since 11.x drops `default.version`. Verified in-browser: 4 diagrams render, `naturalWidth > 0`, valid XML, console reports mermaid 11.15.0.
